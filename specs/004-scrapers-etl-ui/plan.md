## Summary

Fix dead scrapers, add ETL HTTP trigger + Telegram commands, extend `JobStatus` to 7 values with matching Kanban UI, and add "Scan Market" button. All on 1 GB RAM VPS — no headless browsers, no new heavy deps.

## Technical Context

**Language/Version**: TypeScript 5.7, Node.js 22, ESM

**Primary Dependencies**: Fastify 5, Telegraf 4, oracledb 6, React 19, @dnd-kit — no new runtime deps needed (cheerio NOT required — scrapers use JSON APIs)

**Storage**: Oracle Autonomous DB (existing schema, no migrations)

**Testing**: Vitest 4 — existing 32 tests must remain green; update scraper tests for new payload shapes

**Target Platform**: Docker on Oracle Always Free (1 GB RAM)

**Project Type**: Monorepo web service + frontend SPA

**Performance Goals**: ETL non-blocking (spawned child process); Fastify handler returns in <50ms

**Constraints**: NO headless browsers; no new npm packages with native bindings; total RAM budget ~400 MB for Node process

**Scale/Scope**: Single-user app; ~500 jobs per ETL run expected
