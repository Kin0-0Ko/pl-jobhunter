# Implementation Plan: Advanced Matching Engine, LLM Isolation, Strict Error Fallbacks, and Cron Optimization

**Branch**: `006-advanced-matching-llm-hardening` | **Date**: 2026-06-29 | **Spec**: [spec.md](spec.md)

## Summary

Harden the ETL pre-filter with seniority title rejection and experience-year regex parsing, add a cross-training wildcard bypass, fix the Ollama prompt to produce objective third-person company metadata (model: `qwen2.5:0.5b`), enforce a null-summary fallback before every Oracle insert (eliminating ORA-01400), and cut the cron interval from 6h to 3h.

### Task 2 — apps/backend/src/ai/ollama.ts: full hardening

**2a. Add constants** (after existing `NEGATIVE_KEYWORDS`):
```ts
const SENIOR_TITLE_KEYWORDS = ['senior', 'lead', 'principal', 'staff', 'architect'];
const JUNIOR_MID_SENIORITY = new Set(['junior', 'mid', 'trainee', 'intern']);

const CROSS_TRAINING_PHRASES = [
  'no previous', 'no prior', 'willing to cross-train', 'cross-train',
  'we can teach', 'open to retraining', 'we will train', 'can teach you',
];

const EXPERIENCE_RE = /(\d+)\+?\s*(?:years?|lata?|lat)\s*(?:of\s+)?(?:experience|doświadczenia)?/gi;
```

**2b. Add profile param to `isRelevantJob()`**:
```ts
export function isRelevantJob(
  job: Job,
  profile?: { target_seniority?: string[]; max_experience_years?: number }
): boolean
```

Logic order:
1. **Seniority check** (title only): if `profile?.target_seniority` is non-empty and every value is in `JUNIOR_MID_SENIORITY`, and title contains any `SENIOR_TITLE_KEYWORDS` → return `false`
2. **Experience check** (description only, skip if no description or no `max_experience_years`): run `EXPERIENCE_RE` over description, collect all matches, take max; if max > `max_experience_years` → check wildcard override first
3. **Cross-training wildcard** (description only): if description contains any `CROSS_TRAINING_PHRASES` → return `true` (bypasses keyword check)
4. **Keyword check** (existing logic, unchanged)

**2c. Replace `buildPrompt()`**:
```ts
function buildPrompt(job: Job, userProfile: string): string {
  const descSection = job.description
    ? `\n\nJob posting:\n${job.description.slice(0, 1500)}`
    : '';

  return `You are a metadata extraction tool. Extract facts about what the company requires.
Do NOT write in first person. Never say "I am", "I have", or "I can".
Output ONLY valid JSON. No markdown, no <think> tags, no explanation text.

User skills: ${userProfile}
Job: ${job.title} at ${job.company}${descSection}

Extract these fields about the COMPANY's requirements:
- match_score: integer 0-100 (how well user skills match this posting)
- summary: one sentence describing what the company seeks, written in third person
- tech_stack: array of technology strings explicitly named in the posting (empty array if none)

Return exactly: {"match_score":<int>,"summary":"<string>","tech_stack":[<strings>]}`;
}
```

**2d. Null summary guardrail in `callOllama()`** — after the existing field validation:
```ts
if (!parsed.summary || parsed.summary.trim() === '') {
  parsed.summary = 'Metadata extraction failed - pending manual review';
}
```

**2e. Confirm `OLLAMA_MODEL` default** is `'qwen2.5:0.5b'` (currently hardcoded as `'qwen3:5b'` fallback — change default):
```ts
const model = process.env.OLLAMA_MODEL ?? 'qwen2.5:0.5b';
```
